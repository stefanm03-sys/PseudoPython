PseudoPy Guides

1.0 - The Basics

1.1 - Comments

Comments write out your thoughts and are ignored by the program. You can write them on one line, or span across numerous.

Single line comments start with a hashtag # or // and continue until the end of the line.

# This is a single line comment in Pseudopy!
// This is also a single line comment in Pseudopy!

Multi line comments can use /* ... */ or line-delimited // ... //.

/*
This is a multi line comment in Pseudopy!
*/

//
This is also a multi line comment in Pseudopy!
It ends when you put another // on a new line.
//
1.2 - Data Types

Data types are unique, but consistent in languages. The four common ones are strings, integers, floats, and booleans.

"This is a string!"
3
3.14
true
1.3 - Syntax

PseudoPy aims for readable syntax. Example patterns:

# This is a comment.

do 1 + 1
do (1 + 2) * 3

var x is 1
var y is 2

repeat amt(2)
    state("Hello!")
endpt

repeat when x == 10 do {
    state(x)
}

is 1 + 1 == 2
when 1 > 0 or 2 > 0 do
    state("true branch")
endpt
1.4 - Variables and Assignment

Variables in PseudoPy are created using var and is. Declare with var, then assign a value.

var x is 10
var name is "Alice"
var sum is 5 + x
state(sum)

Functions are also creatable using function and closed with endpt.

function greet(user)
    state("Hello there!")
endpt
