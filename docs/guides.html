1.0 - The Basics

Use this page from top to bottom. Each step builds on the previous one while keeping the original guide explanations.

1.1 - Comments

Comments write out your thoughts and are ignored by the program. You can write them on one line, or span across numerous.

Single line comments start with a hashtag # or // and continue until the end of the line.

# This is a single line comment in Pseudopy!
// This is also a single line comment in Pseudopy!

Multi line comments can use /* ... */ or line-delimited // ... //. 

/*
This is a multi line comment in Pseudopy!
*/

//
This is also a multi line comment in Pseudopy!
It ends when you put another // on a new line.
//

1.3 - Syntax Basics

PseudoPy aims for readable syntax and easy coding. Example patterns:

# This is a comment!

do 1 + 1 # This is a standard math expression.
do (1 + 2) * 3 # This a math expression with order of operations.

var x is 1 # This is a variable declaration and assignment horizontally.
var y is
    2 # This is a variable declaration and assignment vertically.

repeat amt(2) # This is code using indentation with an endpt.
    state("Hello!")
endpt

repeat when x == 10 do { # This is code using braces and no endpt.
    state(x)
}

is 1 + 1 == 2 # This is an operation that evaluates to a boolean value. This is true, so it evaluates to true.

when 1 > 0 or 2 > 3 do # The or operator evaluates to true if either side is true, so this condition is true.
    state("true branch")
endpt
1.4 - Variables and Assignment

Variables in PseudoPy are created using var and is. Declare with var, then assign a value using is. They can be called by name later in the program, and can generally be reassigned to new values.

var x is 10 # Creates a variable called x and assigns it the value of 10, which is an integer.
var name is "Alice" # Creates a variable called name and assigns it the value of "Alice", which is a string.
var sum is 5 + x # Creates a variable called sum and assigns it the value of 5 + x, which is 15.
state(sum) # Prints the variable sum, which is 15.
1.5 - Functions

Functions are also creatable using function and closed with endpt. Using this creates a shiny new function that can be customized to your needs. 

function greet(user) # This is a function that takes in a parameter called user.
    state("Hello there!") # These are the content of the function, which will run when the function is called.
endpt # This simply closes the code.
